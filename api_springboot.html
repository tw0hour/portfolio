Ms_Push

Contexte :

C’était durant ma deuxième année d’alternance à Carrefour (2021/2022) j’étudias à ce moment-là à encore à l’ESGI. Cette année mon tuteur m’avait changé d’équipe, dorénavant je devais travailler avec une équipe qui s’occuper de gérer le panier d’achat sur le site Carrefour.com le stack technique utilisé était majoritairement du Java SpringBoot. L’application sur laquelle est quant à elle une application interne qui servirai à l’usage de l’équipe. J’ai travaillé dessus pendant toute ma deuxième année d’alternance. J’étais le seule travailler dessus.


Description du projet :

Ms_Push est une Api de mailing qui reçois une requête en entrée et qui selon le contenu de la requête renvoie un email contenant une page HTML avec un contenu adapté. Le corps de la requête devait contenir une liste de destinataires représenté par des adresses mail, un template représentant une donnée qui me servira à choisir quel html généré, et un body qui représente des données descriptives. Je devais dans un premier temps filtrer toutes les adresses mail et selon certains critères les catégoriser comme étant importants ou non importants. Une fois cela fait je devais vérifier la valeur du Template pour générer un html adapté en insérant les données descriptives. Enfin je devais renvoyer cet HTML sous forme de mails groupé à toutes les destinataires.


Stack technique :

Cette application a été faite en Java 11 (Langage de programmation très utilisé en entreprise) avec le framework SpringBoot. J’ai utilisé le moteur de template de spring pour générer mes html. J’ai également implémenté des tests unitaires / et d’intégrations en utilisant les librairies de java : junit et mockito. Cette application devait être architecturé de la manière suivante : Application, Domain Infrastructure. Ce type d’architecture est appelé l’architecture hexagonal. C’est une architecture qui rend les applications plus flexibles, modulaires et faciles à maintenir.

  • • Le domaine correspond à la partie principale du système, celle qui gère les règles et les éléments spécifiques au métier.
  • • L'application coordonne les différentes parties du système et permet aux utilisateurs d'interagir avec celui-ci.
  • • L'infrastructure s'occupe des aspects techniques comme le stockage des données et les interfaces utilisateur.
