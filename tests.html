Test unitaires / d’intégration / fonctionnel


Les tests logiciels sont une pratique courante dans le développement, que ce soit en Java, en JS ou dans d'autres langages. Au fil de mes années d'alternance, j'ai eu l'opportunité d'écrire de nombreux tests, à la fois à l'école et en entreprise. Cette expérience m'a permis d'acquérir une expertise solide dans ce domaine.


Les tests unitaires

Les tests unitaires permettent de tester chaque composant logiciel de manière isolée, en évaluant son comportement de manière individuelle. J'ai pu mettre en place des tests qui vérifient le bon fonctionnement des fonctions, et au besoin, j'ai utilisé des "mocks" pour simuler les dépendances nécessaires à l'exécution de ces fonctions.


Les tests d'intégration

Les tests d'intégration visent à évaluer le comportement d'un ensemble de composants interconnectés les uns aux autres. En général, ils sont utilisés pour tester des routes d'une API. Ces tests permettent de s'assurer que toutes les parties du système fonctionnent correctement ensemble.


Les tests fonctionnels

Les tests fonctionnels, également appelés tests utilisateurs, sont des tests qui peuvent être réalisés directement sur le logiciel par les utilisateurs eux-mêmes. Ils consistent simplement à tester les fonctionnalités du logiciel telles qu'elles sont présentées à l'utilisateur. Il existe des outils qui permettent d'automatiser ces tests. Dans mon expérience professionnelle, j'ai utilisé l'outil Cypress pour effectuer ces tests.


Conclusion

Les tests logiciels revêtent une importance cruciale dans le développement d'applications. Les tests unitaires, les tests d'intégration et les tests fonctionnels jouent chacun un rôle spécifique dans l'assurance de la qualité du logiciel. Grâce à mon expérience dans l'écriture de ces tests, j'ai pu développer une expertise solide dans ce domaine essentiel.
